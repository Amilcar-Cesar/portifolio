# API de entregas

## O problema
A gestão de rotas de entregas, atribuição de pedidos a motoristas e autenticação segura de motoboys e clientes em trânsito exige uma solução segura e de baixa latência. O objetivo do projeto foi construir uma API demonstrando arquitetura limpa com controle de acessos flexível baseado em JSON Web Tokens (JWT) para garantir que apenas entregadores autorizados visualizem e aceitem rotas de entrega.

## Stack tecnológica e decisões
- **Flask**: Framework leve que permite organizar o projeto utilizando o padrão MVC e estender funcionalidades sem inchaço de pacotes.
- **JWT (PyJWT)**: Autenticação stateless baseada em tokens. Essencial para aplicações móveis de entrega, pois dispensa sessões no servidor e lida facilmente com redes instáveis.
- **SQLite**: Banco de dados relacional em arquivo local, perfeito para um ambiente de desenvolvimento rápido e demonstração leve e portável do projeto.

## Desafio técnico e solução
**O desafio: interceptação de rotas e segurança de acessos baseada em papéis (Role-Based Access Control)**
Garantir que um entregador não possa acessar as rotas atribuídas a outro entregador, e que clientes não possam alterar o status de entrega das ordens.

**A solução:**
Criação de um decorador de autenticação personalizado em Flask para verificar e decodificar o token JWT recebido no cabeçalho `Authorization`, extraindo a identidade e a role (papel) do usuário:

```python
# decorators.py
from functools import wraps
from flask import request, jsonify
import jwt
from config import SECRET_KEY

def token_required(allowed_roles=None):
    def decorator(f):
        @wraps(f)
        def decorated(*args, **kwargs):
            token = None
            if 'Authorization' in request.headers:
                auth_header = request.headers['Authorization']
                if auth_header.startswith("Bearer "):
                    token = auth_header.split(" ")[1]

            if not token:
                return jsonify({'message': 'Token ausente.'}), 401

            try:
                data = jwt.decode(token, SECRET_KEY, algorithms=["HS256"])
                current_user = data['user_id']
                user_role = data.get('role')
            except Exception:
                return jsonify({'message': 'Token inválido ou expirado.'}), 401

            if allowed_roles and user_role not in allowed_roles:
                return jsonify({'message': 'Acesso não autorizado.'}), 403

            return f(current_user, *args, **kwargs)
        return decorated
    return decorator
```
Esse mecanismo garantiu controle de acesso centralizado e proteção robusta das rotas de entrega.
