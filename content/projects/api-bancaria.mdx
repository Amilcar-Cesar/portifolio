# API bancária minimalista em Python

Uma API bancária minimalista escrita em Python, criada como projeto de estudo e aplicação de portfólio. O objetivo principal é demonstrar organização em arquitetura MVC, clareza no design de código e a capacidade de implementar operações bancárias básicas (criação de contas, listagem, saques e extratos) usando MySQL como persistência principal.

> **Status:** Projeto de estudo / portfólio — não destinado a produção.

## Visão Geral & Propósito
- **Propósito:** Exibir habilidades de arquitetura, organização de código e testes básicos.
- **Arquitetura:** Model-View-Controller (MVC) com inversão de dependência através de interfaces.
- **Persistência:** MySQL (container local via `docker-compose.yml` / esquema em `init/schema.sql`).
- **Operações básicas:** Criação de contas, listagem, saques e extratos.

## Stack tecnológica e decisões
- **Python**: Linguagem principal para estruturar a arquitetura MVC de forma modular e testável.
- **Interfaces & Inversão de Dependência**: Uso de interfaces abstratas para desacoplar a camada de controller da persistência MySQL.
- **MySQL & SQLAlchemy**: Banco relacional para persistência dos dados de clientes e saldos com esquema em `init/schema.sql`.
- **Docker & Docker Compose**: Conteinerização da aplicação e banco de dados via `docker-compose.yml`.

## Lógica de Negócio e Validações

### O desafio: garantia de regras de negócio e tratamento de erros
Em operações bancárias básicas de saque e extrato, é essencial validar a existência do cliente, verificar se há saldo suficiente, impedir saques com valores negativos ou nulos e aplicar o limite máximo permitido por transação, retornando erros HTTP apropriados caso qualquer regra seja violada.

### A solução: validações modulares em Controller com exceções HTTP
Implementação da classe `CpfSaqueExtratoController` desacoplada do repositório por meio da interface `CpfRepositoryInterface`. O fluxo de saque realiza validações isoladas e lança exceções HTTP customizadas (`HttpNotFound`, `HttpForbidden`, `HttpBadRequest`):

```python
from src.models.mysql.interface.cpf_repository import CpfRepositoryInterface
from src.controller.interface.cliente_interface import ClienteInterface
from src.errors.types.http_not_found import HttpNotFound
from src.errors.types.http_bad_request import HttpBadRequest
from src.errors.types.http_forbiden import HttpForbidden

class CpfSaqueExtratoController(ClienteInterface):
    def __init__(self, cpf_repository: CpfRepositoryInterface) -> None:
        self.__cpf_repository = cpf_repository

    def saque(self, cliente_id: int, valor_saque: float) -> dict:
        self.__validar_cliente(cliente_id)
        saldo_result = self.__saldo_suficiente(cliente_id, valor_saque)
        self.__limite_saque(valor_saque)

        novo_saldo = saldo_result["novo_saldo"]
        self.__cpf_repository.atualizar_saldo(cliente_id, novo_saldo)
        
        return {"sucesso": True, "novo_saldo": novo_saldo}

    def __validar_cliente(self, cliente_id: int):
        account = self.__cpf_repository.get_account(cliente_id)
        if not account:
            raise HttpNotFound("Conta não encontrada.")
        return account

    def __saldo_suficiente(self, cliente_id: int, valor_saque: float):
        saldo_atual = self.__cpf_repository.get_saldo(cliente_id)

        if saldo_atual < valor_saque:
            raise HttpForbidden("Saldo insuficiente")

        if valor_saque <= 0:
            raise HttpBadRequest("Valor deve ser maior que zero")

        return {"sucesso": True, "novo_saldo": saldo_atual - valor_saque}

    def __limite_saque(self, valor_saque: float):
        if valor_saque > float(1000.0):
            raise HttpForbidden("Valor de saque excede o limite!")
        return None

    def extrato(self, cliente_id: int) -> dict:
        self.__validar_cliente(cliente_id)
        saldo = self.__cpf_repository.get_saldo(cliente_id)

        return {"mensagem": "Saldo atual da conta", "saldo": saldo}
```

Essa estrutura garante clareza de responsabilidades no padrão MVC, testabilidade unitária e tratamento padronizado de exceções.
